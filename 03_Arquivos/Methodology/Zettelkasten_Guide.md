# Guia: Método Index Card para Programadores

Este guia adapta o método Zettelkasten/Index Cards para o estudo de programação.

## Fluxo de Trabalho

### 1. Captura (Fleeting Notes) - `00_Entrada`
Quando surgir uma ideia ou dúvida rápida, crie um arquivo na pasta `00_Entrada`.
- **Formato**: Rápido, sem estrutura rígida.
- **Tag**: `#Inbox`

### 2. Processamento (Literature/Permanent Notes)
Periodicamente, revise sua `00_Entrada`. Para cada nota que vale a pena guardar:
1. Mova para a pasta apropriada em `02_Areas` (ex: `02_Areas/React`).
2. Aplique o template **Index Card** (`03_Arquivos/Templates/IndexCard.md`).
3. Preencha as seções: Definição, Como Funciona, Exemplo.
4. **Conecte**: Adicione links para outras notas relacionadas.

### 3. Projetos - `01_Projetos`
Se várias notas se juntam para formar uma ideia de aplicativo ou ferramenta:
1. Crie uma pasta em `01_Projetos`.
2. Mova ou linke as notas relevantes.
3. Comece a codar!

## Estrutura do Index Card
- **O que é?**: Definição simples.
- **Como funciona?**: O "debaixo do capô".
- **Código**: Snippets que você pode copiar e colar.
- **Por que usar?**: Trade-offs e casos de uso.
